# 🧩 Templates Guide

Create powerful, reusable page structures with Filament CMS templates. Templates define both the form fields editors see and how content appears on your website.

## 🎯 Understanding Templates

### What Are Templates?

Templates are PHP classes that define:
- **Form fields** for content editors
- **Data structure** for page content  
- **Rendering logic** for frontend display
- **Validation rules** for content

### Template Concept

> 💡 Based on [Dennis Koch's Filament Trick](https://filamentphp.com/tricks/template-based-forms) for dynamic form generation.

**Example workflow:**
1. **Editor selects** "Blog Post" template
2. **Form shows** title, content, featured image, tags
3. **Content is saved** in structured format
4. **Frontend renders** using blog post layout

## 🚀 Creating Templates

### Quick Template Generation

```bash
php artisan filament-cms:page-template BlogPost
```

This creates:
- Template class: `App\CmsPages\Templates\BlogPost`
- View file: `resources/views/cms/templates/blog-post.blade.php`

### Basic Template Structure

```php
<?php

namespace App\CmsPages\Templates;

use Filament\Forms;
use SolutionForest\FilamentCms\CmsPages\Contracts\CmsPageTemplate;
use SolutionForest\FilamentCms\CmsPages\Renderer\AtomicDesignPageRenderer;

final class BlogPost implements CmsPageTemplate
{
    public static function title(): string
    {
        return 'Blog Post';
    }

    public static function schema(): array
    {
        return [
            Forms\Components\Card::make()
                ->schema([
                    Forms\Components\TextInput::make('subtitle')
                        ->label('Subtitle')
                        ->maxLength(255),
                        
                    Forms\Components\RichEditor::make('content')
                        ->label('Post Content')
                        ->required(),
                        
                    Forms\Components\FileUpload::make('featured_image')
                        ->label('Featured Image')
                        ->image()
                        ->maxSize(2048),
                        
                    Forms\Components\DatePicker::make('publish_date')
                        ->label('Publish Date')
                        ->default(now()),
                        
                    Forms\Components\TagsInput::make('tags')
                        ->label('Tags')
                        ->placeholder('Add tags...'),
                ]),
        ];
    }

    public static function getRenderer(): ?string
    {
        return AtomicDesignPageRenderer::class;
    }

    public static function hiddenOnTemplateOptions(): bool
    {
        return false; // Set to true to hide from template selector
    }
}
```

## 🔧 Advanced Template Features

### Custom Validation

Add validation rules to your templates:

```php
public static function schema(): array
{
    return [
        Forms\Components\TextInput::make('email')
            ->label('Contact Email')
            ->email()
            ->rules(['required', 'email', 'unique:cms_pages,data->email']),
            
        Forms\Components\TextInput::make('price')
            ->label('Price')
            ->numeric()
            ->minValue(0)
            ->rules(['required', 'numeric', 'min:0']),
    ];
}
```

### Conditional Fields

Show/hide fields based on other field values:

```php
Forms\Components\Select::make('type')
    ->label('Content Type')
    ->options([
        'article' => 'Article',
        'video' => 'Video',
        'gallery' => 'Gallery',
    ])
    ->reactive(),

Forms\Components\TextInput::make('video_url')
    ->label('Video URL')
    ->url()
    ->visible(fn ($get) => $get('type') === 'video'),

Forms\Components\Repeater::make('images')
    ->label('Gallery Images')
    ->schema([
        Forms\Components\FileUpload::make('image')->image(),
    ])
    ->visible(fn ($get) => $get('type') === 'gallery'),
```

### Grouped Fields

Organize complex forms with sections:

```php
public static function schema(): array
{
    return [
        Forms\Components\Tabs::make('Content')
            ->tabs([
                Forms\Components\Tabs\Tab::make('Basic Info')
                    ->schema([
                        Forms\Components\TextInput::make('title')->required(),
                        Forms\Components\Textarea::make('description'),
                    ]),
                    
                Forms\Components\Tabs\Tab::make('SEO')
                    ->schema([
                        Forms\Components\TextInput::make('meta_title'),
                        Forms\Components\Textarea::make('meta_description'),
                    ]),
                    
                Forms\Components\Tabs\Tab::make('Advanced')
                    ->schema([
                        Forms\Components\KeyValue::make('custom_fields'),
                    ]),
            ]),
    ];
}
```

## 🎭 Template Renderers

### Default Renderer

The `AtomicDesignPageRenderer` is the default renderer that works with your theme files.

### Custom Renderers

Create custom renderers for advanced template logic:

```bash
php artisan filament-cms:page-template-renderer CustomRenderer
```

Example custom renderer:

```php
<?php

namespace App\CmsPages\Renderers;

use SolutionForest\FilamentCms\CmsPages\Contracts\Renderer;
use SolutionForest\FilamentCms\Dto\CmsPageData;

class CustomRenderer implements Renderer
{
    public function render(CmsPageData $page, array $layout): string
    {
        // Custom rendering logic
        $data = $page->data;
        
        // Process template data
        $processedData = $this->processTemplateData($data);
        
        // Return rendered HTML
        return view('custom.template', [
            'page' => $page,
            'data' => $processedData,
            'layout' => $layout,
        ])->render();
    }
    
    private function processTemplateData(array $data): array
    {
        // Add custom processing logic
        return $data;
    }
}
```

Use custom renderer in your template:

```php
public static function getRenderer(): ?string
{
    return CustomRenderer::class;
}
```

## 🎯 Template Best Practices

### Planning Templates

1. **Identify content types** your site needs
2. **Map out required fields** for each type
3. **Consider editor experience** when designing forms
4. **Plan for future expansion** with flexible structures

### Field Organization

- **Group related fields** together
- **Use descriptive labels** and help text
- **Order fields logically** (most important first)
- **Use appropriate field types** for data

### Performance Considerations

- **Limit repeater fields** to reasonable numbers
- **Optimize image uploads** with size limits
- **Use efficient field types** (avoid heavy components)
- **Consider caching** for complex templates

### Maintenance

- **Document custom templates** thoroughly
- **Version control** template changes
- **Test templates** after updates
- **Provide fallbacks** for missing data

### Template Inheritance

Create base templates for common functionality:

```php
abstract class BaseTemplate implements CmsPageTemplate
{
    protected static function getCommonFields(): array
    {
        return [
            Forms\Components\TextInput::make('seo_title')
                ->label('SEO Title'),
            Forms\Components\Textarea::make('seo_description')
                ->label('SEO Description'),
        ];
    }
}

class BlogPost extends BaseTemplate
{
    public static function schema(): array
    {
        return array_merge(
            [
                Forms\Components\RichEditor::make('content')
                    ->label('Post Content'),
            ],
            static::getCommonFields()
        );
    }
}
```

## 🆘 Troubleshooting Templates

### Common Issues

**Template not appearing in selector:**
- ✅ Check `hiddenOnTemplateOptions()` returns `false`
- ✅ Verify class implements `CmsPageTemplate`
- ✅ Clear application cache

**Form validation errors:**
- ✅ Check field validation rules
- ✅ Verify required fields are filled
- ✅ Test field dependencies

**Rendering issues:**
- ✅ Verify renderer class exists
- ✅ Check template view files
- ✅ Test with default renderer

### Testing Templates

```php
// Test template in isolation
$template = new BlogPost();
$schema = $template::schema();
$renderer = $template::getRenderer();
```
